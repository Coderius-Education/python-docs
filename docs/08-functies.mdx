# Functies

Een functie is een blok code dat je een naam geeft. Je schrijft het één keer en kunt het daarna zo vaak hergebruiken als je wilt — zonder alles opnieuw te typen.

---

## Predict

Bekijk de onderstaande code met twee functies. Wat denk je dat er op het scherm verschijnt? In welke volgorde worden de regels geprint?

```python
def lijn():
    print("=" * 20)

def welkom():
    lijn()
    print("  Welkom bij Python!")
    lijn()

print("Ik ga de functies aanroepen:")
welkom()
print("Dat was het!")
```

Schrijf je voorspelling op. Let goed op de volgorde!

---

## Run

Zet de code in de [Playground](/playground) en voer het uit.

<details>
<summary>👀 Kwam dit overeen met je voorspelling?</summary>

```text
Ik ga de functies aanroepen:
====================
  Welkom bij Python!
====================
Dat was het!
```

Python voert de code van boven naar beneden uit. Maar de regels *binnen* een `def` worden pas uitgevoerd als je de functie verderop **aanroept** (met de haakjes `()`). De functie `welkom()` roept zelf ook weer de functie `lijn()` aan — functies mogen functies aanroepen!

</details>

---

## Investigate

1. Wat gebeurt er als je `welkom()` een tweede keer aanroept, nóg een keer onder `"Dat was het!"`? Dat is precies de kracht van functies: schrijf het één keer, gebruik het oneindig vaak.
2. Verwijder de haakjes `()` achter `welkom` op de aanroepregel. Wordt er nu nog iets geprint? Waarom niet?
3. Wat is het verschil tussen het **definiëren** (`def welkom():`) en het **aanroepen** (`welkom()`) van een functie?

:::caution Let op
De code binnen een functie móet inspringen (4 spaties), net als bij `if` en `for`. Zonder inspringen krijg je een `IndentationError`.
:::
