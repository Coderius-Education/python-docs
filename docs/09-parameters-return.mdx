# Parameters en Return

In de vorige les maakten we functies die altijd hetzelfde doen. Nu leren we functies maken die informatie **ontvangen** (parameters) en een **resultaat teruggeven** (return).

---

## Predict

Bekijk de onderstaande code. Wat denk je dat er op het scherm verschijnt? Let goed op de variabele `resultaat` — welke waarde heeft die?

```python
def verdubbel(getal):
    return getal * 2

resultaat = verdubbel(5)
print(f"Het resultaat is: {resultaat}")
print(f"Zes verdubbeld is: {verdubbel(6)}")
```

Schrijf je voorspelling op voordat je verder leest.

---

## Run

Zet de code in de [Playground](/playground) en voer het uit.

<details>
<summary>👀 Kwam dit overeen met je voorspelling?</summary>

```text
Het resultaat is: 10
Zes verdubbeld is: 12
```

De functie `verdubbel` ontvangt een **parameter** genaamd `getal`. Wanneer je `verdubbel(5)` aanroept, wordt `getal` automatisch `5`. Vervolgens geeft `return getal * 2` de waarde `10` terug, die je dan kunt opslaan in `resultaat`.

</details>

## Investigate

Kopieer de volgende code naar de Playground en beantwoord de vragen:

```python
def optellen(a, b):
    return a + b

def begroet(naam):
    print(f"Hallo, {naam}!")

som = optellen(3, 5)
print(som)

x = begroet("Alex")
print(x)
```

1. Welke waarde heeft `som`? En welke waarde heeft `x`? Voer de code uit.
2. Waarom is `x` gelijk aan `None`? Wat is het verschil tussen `print()` en `return`?
3. Wat gebeurt er als je `optellen(3, 5)` aanroept maar het resultaat **niet** opslaat in een variabele? Probeer: verwijder de regel `som = ...` en typ alleen `optellen(3, 5)`. Verschijnt er iets op het scherm?

:::caution print() vs return
`print()` toont iets op het scherm, maar geeft **niks** terug. `return` geeft een waarde terug die je kunt **opslaan** en later **hergebruiken**. Dit is een van de meest gemaakte fouten door beginners!
:::
